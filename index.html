エージェント実務化
Claude Codeを使いたい。でも、実務でどう使えばいいか分からない。そこに、いなじろうが入ります。
Problem

止まっているのは、ツールの知識ではなく実務への入れ方です。

「便利そう」は分かっている。でも、チャットで聞くだけでは仕事の流れは変わらない。Claude Codeをどの業務にどう入れるかを決める必要があります。

知識で終わるClaude Codeの概要を聞いても、翌日から任せる作業が決まっていない。
個人利用で止まる詳しい人だけが触り、社内の共通手順や確認ルールにならない。
仕事が変わらないチャットの回答は増えても、ファイル修正・資料整備・確認作業の流れに入っていない。
Approach

最初に作るべきは、Claude Codeで仕事が進む体験です。

いきなり全社展開しません。まずは小さな1業務で、AIエージェントがファイルを読み、直し、人が確認する流れを作ります。

01

見つける

Claude Codeに任せやすい作業と、NotebookLMなど別ツールが向く作業を切り分けます。

02

見せる

ファイルを読ませ、修正させ、差分を確認する流れを実演・体験します。

03

残す

AGENTS.mdやSkillsなど、繰り返し使えるルール化まで見せます。

First Use Case

入口は、ファイルや資料がある身近な業務でいい。

大きなシステム刷新ではなく、LP修正、手順書整理、議事録整理、問い合わせ対応のように、担当者が効果を感じやすい業務から始めます。

今の状態

  • 資料・文章・ファイル修正に時間がかかる
  • 人によって手順や品質がバラつく
  • 過去の資料や手順が使い回せていない

最初に見せたい変化

  • Claude Codeで修正・整理・差分確認が速くなる
  • 人が差分を確認する前提で安心して使える
  • 「次の業務も任せられそう」が生まれる
